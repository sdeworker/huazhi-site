## 加一篇新文章的标准流程

用户会给一篇 `.docx`（通常是从微信公众号导出的）。步骤：

1. **提取内容**：`pandoc -t markdown 文章.docx -o /tmp/art.md`

2. **清洗**——公众号导出的正文里塞满了垃圾，**必须全部删掉**：
   - 「长按下图可关注」「点右上角…关注」
   - 「小编微信：XXX 手机号」← **私人手机号绝不能上公网**
   - 「预览时标签不可点」「微信扫一扫」「使用小程序」
   - 所有 `javascript:void(0)` 链接
   - 公众号二维码图片、作者头像图片

3. **复制一个现有文章页当模板**（`sanshijiao-02-bianhua.html` 结构最全），改：
   - `<title>` 和 `<meta description>`
   - `.art-title`、`.art-meta`（作者/日期/编号）
   - 正文
   - 页脚的 `.art-nav`（上一篇/下一篇链接）

4. **命名**：`sanshijiao-NN-拼音短名.html`（如 `sanshijiao-03-yijing.html`）。**只用小写 ASCII，不要中文文件名，不要空格。**

5. **在 `articles.html` 里加一张卡片**，并把 `.list-head .cnt` 的篇数更新掉。

6. **改上一篇的"下一篇"链接**指向新文章（保持文章之间的串联）。

7. `git add . && git commit && git push` → 1 分钟后自动上线。

### 排版可用的组件（都在 article.css 里定义好了）

- `.lead-box` — 文首「本文导读」问题框
- `.pull` — 重点引语框（挑作者最锋利的那几句，别滥用，一篇 3–5 个）
- `.fig` + 内联 SVG — 图示。**优先重画成 SVG**，不要用原 docx 里的模糊 jpg（矢量清晰、免传图片文件、自动配色）
- `<h2>` 大节 / `<h3><span class="ix">01</span>标题</h3>` 小节
- `<em>` 会渲染成青色强调（非斜体）

## 视觉与内容红线

### 首页的「纠缠球」

`index.html` 的 hero 是一个 canvas 手写的 3D 粒子球（约 900 个点，Fibonacci 球面分布）。**它不是装饰**，它是 SDE 的可视化：

> 光点沉睡在 E（纠缠）中 → 鼠标是「显露焦点」→ 靠近处光点被点亮、连线、结晶为 S（显露）→ 移开则沉回混沌。
> 即 **在 E 中 → 经 D → 成 S**。

改动它时不要破坏这个语义。纯 Canvas 2D，**零外部依赖**，不要引入 three.js。

### SDE 术语（权威定义，2026-07《SDE本体论入门》）

网站正文里出现 SDE 概念时必须准确：

- **SDE = 显露—差异—纠缠**。S = **Show 显露**，**不是** Structure 结构。「结构—差异—纠缠」是已废弃的早期叫法。
- **S（显露）** 是一条从弱到强的连续谱，结构只是它最坚硬的一端。
- **E（纠缠）** 是参与发生的**土壤/底盘**，不是被动的「背景」。
- **D（差异）** 关心路径如何生成，不是中性的「步骤」或「变化」。
- **27 宫格** = C 规范轴（对比·变化·分布）⊗ M 模态轴（粒子·波·场）⊗ V 价值轴（真·善·美）。
- **「发生」不能写成「产生」**。这是核心区分。

> 「三视角智慧」专栏（王德生 2017）里的 **对比·变化·分布 = C 规范轴**，**粒子·波·场 = M 模态轴**。
> 这个专栏是 SDE 的**源头形态**，不是「旧文归档」。专栏页上已经这样定位，保持住。

### 内容纪律

- **公众号文章原文尽量少改**——用户写作是为了自己生命的梳理和思想的真实表达，不是讨好读者。**不要为了「让人看懂」而软化、简化或牺牲思想的锋芒。** 可以做的只有：分节、加小标题、挑引语、修排版。
- 智能体产品名单（index.html 底部那六张卡）目前是**草稿**，用户尚未最终确认对外名称。改动前先问。

## 技术约束

- **不要引入构建工具**（webpack/vite/npm）。Cloudflare 那边 build command 是**空的**，加了会挂。
- **不要引入前端框架**。纯 HTML + CSS + 原生 JS。
- **不要用 localStorage/sessionStorage**。
- 外部依赖只有一个：Google Fonts（Sora / Inter / Space Mono / Noto Serif SC）。
- 深色主题。配色变量在 `article.css` 的 `:root` 和 `index.html` 的 `<style>` 里**各存了一份**——改配色时**两处都要改**。
  - `--E: #6D5EF6`（纠缠·紫）`--D: #FF5C8A`（差异·品红）`--S: #2DE1C2`（显露·青）
- 移动端必须能用（导航会收成汉堡菜单，纠缠球点数自动降到 400）。
- 尊重 `prefers-reduced-motion`。
